# Agent Instructions — personal-platform-infra

Instruções diretas para AI agents. Lidas com prioridade maior que CLAUDE.md pelo OpenCode.
Para contexto do projeto (estrutura, serviços, comandos), ver CLAUDE.md.

## Regras absolutas

- Nunca commitar direto na `main` — sempre branch + PR via `gh pr create`
- Nunca rodar `kubectl delete` sem confirmar o contexto (VPS vs local)
- Nunca expor secrets em plaintext — usar SOPS (`secrets/*.enc.yaml`) ou `{env:VAR}`
- Sempre rodar `kubectl kustomize` antes de propor mudança em `k8s/`
- Nunca usar Helm — Kustomize apenas (ADR 0007)
- Nunca colocar storage dentro do cluster sem PVC explícito (ADR 0002)
- Nunca usar image tag `latest` em manifests — sempre tag específica ou `main`

## Fluxo para mudança em manifests k8s

1. `explorer` — entender estado atual (portas, health paths, overlays existentes)
2. `infra-engineer` — editar base e/ou overlay
3. Validar: `kubectl kustomize k8s/overlays/local` e `kubectl kustomize k8s/overlays/vps`
4. `reviewer` — checar segurança, ADRs, sintaxe antes de abrir PR

## Fluxo para scripts e automação

1. `scripter` — criar ou editar em `scripts/`
2. Validar: `bash -n <script>.sh` ou parse PowerShell
3. Adicionar recipe no `Justfile` se for novo script
4. Smoke test antes de commitar

## Fluxo para mudança no gateway

Qualquer mudança em `GATEWAY_TOOL_ALLOWLIST` exige verificar os nomes reais das tools
no upstream MCP correspondente — o gateway valida o catálogo estático na inicialização
e recusa tools desconhecidas. Confirmar com `curl` ou MCP protocol call antes de editar.

## Ambientes e contextos

- **Local (k3d):** contexto kubectl `k3d-*` ou `default` — pode testar livremente
- **VPS (k3s):** contexto kubectl com `vps`/`k3s`/`prod` no nome — operações destrutivas requerem confirmação explícita

O plugin `context-guard.ts` anota automaticamente bash commands com `[VPS]` ou `[k3d]`
e bloqueia `kubectl delete` destrutivos em contexto VPS.

## Delegação entre agents

Prefira compor agents em vez de fazer tudo em um só:

| Task | Agent recomendado |
|---|---|
| Investigar antes de agir | `explorer` |
| Editar k8s, terraform, ansible | `infra-engineer` |
| Criar/editar scripts, Justfile | `scripter` |
| Revisar antes de merge/deploy | `reviewer` |
| Smoke, logs, wake/sleep | `operations` |
| Coordenar task complexa | `orquestrador` |

## O que nunca delegar ao agent sem revisão humana

- Merge de PR
- Apply de overlay VPS (`kubectl apply` em contexto de produção)
- Edição de secrets SOPS
- Push com `--force`
